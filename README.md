# 우리 반 강점지도 표준 배포판

친구들이 서로의 강점과 이유, 응원 한마디를 남기고 학생 개인별 결과와 관리자 결과를 확인할 수 있는 웹 기반 롤링페이퍼 프로그램입니다.

---

## 1. 파일 구성

| 파일 | 역할 | 학교별 수정 여부 |
|---|---|---|
| `index.html` | 학생용 화면 | 원칙적으로 수정하지 않음 |
| `admin.html` | 관리자용 화면 | 원칙적으로 수정하지 않음 |
| `app.js` | 학생용 기능 코드 | 수정하지 않음 |
| `style.css` | 학생 화면 디자인 | 수정하지 않음 |
| `config.js` | 학교별 설정 파일 | 수정 필요 |
| `Code.gs` | Google Apps Script 백엔드 코드 | `ADMIN_PASSWORD` 수정 권장 |

---

## 2. 다른 학교에서 반드시 수정할 곳

### 2-1. `config.js`

`config.js`의 `[1] 학교별 수정 구역`만 수정합니다.

```js
const SCHOOL_NAME = "○○학교";
const PROGRAM_TITLE = "우리 반 강점지도";
const GAS_URL = "https://script.google.com/macros/s/여기에_웹앱_배포주소를_넣으세요/exec";
const CLASS_LIST = ["1", "2", "3", "4", "5"];
const MAX_STUDENT_NUMBER = 30;
```

| 항목 | 설명 | 예시 |
|---|---|---|
| `SCHOOL_NAME` | 학교명 | `"창원○○고등학교"` |
| `PROGRAM_TITLE` | 화면 제목 | `"우리 반 강점지도"` |
| `GAS_URL` | Apps Script 웹앱 주소 | `/exec`로 끝나는 주소 |
| `CLASS_LIST` | 운영할 반 목록 | `["1", "2", "3"]` |
| `MAX_STUDENT_NUMBER` | 한 반 최대 번호 | `30` |

주의: `GAS_URL`을 바꾸지 않으면 다른 학교 학생의 응답이 기존 학교 구글시트로 저장될 수 있습니다.

---

## 3. 구글시트 명단 양식

구글시트에는 반드시 `명단` 시트가 있어야 하며, 1행은 아래 순서로 작성합니다.

| 반 | 번호 | 이름 | 코드 | 관심진로 |
|---|---|---|---|---|
| 1 | 1 | 김민준 | 4821 | 경찰 |
| 1 | 2 | 이서연 | 7350 | 간호사 |
| 1 | 3 | 박지훈 | 1964 | 컴퓨터공학 |

### 열 설명

| 열 | 내용 |
|---|---|
| A열 `반` | 학생의 반 |
| B열 `번호` | 학생 번호 |
| C열 `이름` | 학생 이름 |
| D열 `코드` | 학생 개인 로그인 코드 |
| E열 `관심진로` | 선택 입력, 비워도 됨 |

명단 시트의 열 이름과 순서는 바꾸지 않는 것을 권장합니다.

---

## 4. Apps Script에서 수정할 곳

`Code.gs` 상단의 관리자 비밀번호만 학교별로 바꾸면 됩니다.

```js
const ADMIN_PASSWORD = '2357';
```

예:

```js
const ADMIN_PASSWORD = '4829';
```

시트명과 헤더는 바꾸지 않는 것을 권장합니다.

---

## 5. 설치 순서

1. 구글시트 새로 만들기 또는 기존 템플릿 복사
2. `명단` 시트에 학생 명단 입력
3. Apps Script에 `Code.gs` 붙여넣기
4. `ADMIN_PASSWORD` 수정
5. Apps Script에서 `setupSheets()` 한 번 실행
6. Apps Script를 웹앱으로 배포
7. 배포 URL을 `config.js`의 `GAS_URL`에 붙여넣기
8. `config.js`의 학교명, 반 목록, 최대 번호 수정
9. GitHub Pages에 파일 업로드
10. 학생 화면 로그인 테스트
11. 관리자 화면 로그인 테스트

---

## 6. 결과 생성 방법

학생들이 모두 작성한 뒤 Apps Script 편집기에서 아래 함수를 실행합니다.

```js
generateResults()
```

그 후 학생은 본인 결과를 볼 수 있고, 관리자는 반 전체 결과를 볼 수 있습니다.

---

## 7. 결과 화면 구조

학생 화면과 관리자 화면은 모두 아래 두 코너로 나뉩니다.

1. `친구들이 발견한 강점과 이유`
   - 강점①
   - 이유①
   - 강점②
   - 이유②

2. `친구들이 남긴 한마디`
   - 한마디만 따로 모아 표시

---

## 8. 입력 제한

이 표준 배포판은 최대 글자 수 제한을 사용하지 않습니다.

| 항목 | 제한 |
|---|---|
| 기타 강점 | 1자 이상 |
| 기타 이유 | 1자 이상 |
| 한마디 | 선택 사항, 비워도 됨 |
| 최대 글자 수 | 제한 없음 |

금칙어 목록은 `config.js`의 `BANNED_WORDS`에서 학교 상황에 맞게 수정할 수 있습니다.

---

## 9. GitHub Pages 반영이 안 될 때

파일을 수정했는데 화면이 바뀌지 않으면 브라우저 캐시 때문일 수 있습니다.

- Windows: `Ctrl + F5`
- 주소 뒤에 `?v=2` 붙이기

예:

```text
index.html?v=2
admin.html?v=2
```

## 10. 학교별 수정 요약표

| 수정 위치 | 수정 내용 | 필수 여부 |
|---|---|---|
| `config.js` `SCHOOL_NAME` | 학교명 | 필수 |
| `config.js` `GAS_URL` | Apps Script 웹앱 주소 | 필수 |
| `config.js` `CLASS_LIST` | 반 목록 | 필수 |
| `config.js` `MAX_STUDENT_NUMBER` | 최대 번호 | 필수 |
| `config.js` `STRENGTH_LIST` | 강점 선택지 | 선택 |
| `config.js` `REASON_LIST` | 이유 선택지 | 선택 |
| `config.js` `BANNED_WORDS` | 금칙어 | 선택 |
| `Code.gs` `ADMIN_PASSWORD` | 관리자 비밀번호 | 권장 |
| 구글시트 `명단` | 학생 명단 | 필수 |
